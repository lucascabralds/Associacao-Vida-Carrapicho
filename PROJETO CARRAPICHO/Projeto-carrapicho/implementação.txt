1. Visão Geral do Sistema
O Sistema ONG Solidária é uma aplicação web completa para gestão administrativa de organizações não governamentais. O sistema permite o gerenciamento de eventos, controle financeiro e administração de usuários em um único painel integrado.
Objetivos
•	Centralizar a gestão de eventos da ONG
•	Controlar receitas e despesas de forma transparente
•	Gerenciar usuários com diferentes níveis de acesso
•	Fornecer relatórios visuais para tomada de decisão

7. Configuração do Backend
7.1 Estrutura do server.js
O arquivo server.js deve conter:
javascript
const express = require('express');
const sqlite3 = require('sqlite3').verbose();
const bcrypt = require('bcryptjs');
const jwt = require('jsonwebtoken');
const multer = require('multer');
const cors = require('cors');
const path = require('path');
const fs = require('fs');

const app = express();
const PORT = 3000;
const JWT_SECRET = 'ong_solidaria_secret_key_2024';

// Middleware
app.use(cors());
app.use(express.json());
app.use('/uploads', express.static(path.join(__dirname, 'uploads')));

// Configuração do banco de dados
const db = new sqlite3.Database('./database.sqlite');

// Configuração de upload de imagens
const storage = multer.diskStorage({
    destination: (req, file, cb) => {
        cb(null, 'uploads/');
    },
    filename: (req, file, cb) => {
        const uniqueSuffix = Date.now() + '-' + Math.round(Math.random() * 1E9);
        cb(null, uniqueSuffix + path.extname(file.originalname));
    }
});

const upload = multer({ 
    storage: storage,
    limits: { fileSize: 5 * 1024 * 1024 }, // 5MB limit
    fileFilter: (req, file, cb) => {
        const allowedTypes = /jpeg|jpg|png|gif/;
        const extname = allowedTypes.test(path.extname(file.originalname).toLowerCase());
        const mimetype = allowedTypes.test(file.mimetype);
        if (mimetype && extname) {
            return cb(null, true);
        }
        cb(new Error('Apenas imagens são permitidas'));
    }
});

// Criar tabelas do banco de dados
db.serialize(() => {
    // Tabela de usuários
    db.run(`CREATE TABLE IF NOT EXISTS users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name TEXT NOT NULL,
        email TEXT UNIQUE NOT NULL,
        password TEXT NOT NULL,
        role TEXT DEFAULT 'editor',
        active BOOLEAN DEFAULT 1,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
    )`);

    // Tabela de eventos
    db.run(`CREATE TABLE IF NOT EXISTS events (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT NOT NULL,
        description TEXT,
        event_date DATETIME NOT NULL,
        location TEXT,
        status TEXT DEFAULT 'upcoming',
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
    )`);

    // Tabela de imagens de eventos
    db.run(`CREATE TABLE IF NOT EXISTS event_images (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        event_id INTEGER NOT NULL,
        image_url TEXT NOT NULL,
        is_cover BOOLEAN DEFAULT 0,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
        FOREIGN KEY (event_id) REFERENCES events(id) ON DELETE CASCADE
    )`);

    // Tabela de transações financeiras
    db.run(`CREATE TABLE IF NOT EXISTS transactions (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        type TEXT NOT NULL,
        category TEXT NOT NULL,
        description TEXT,
        amount REAL NOT NULL,
        transaction_date DATE NOT NULL,
        created_at DATETIME DEFAULT CURRENT_TIMESTAMP
    )`);

    // Criar usuário admin padrão
    const adminPassword = bcrypt.hashSync('admin123', 10);
    db.run(`INSERT OR IGNORE INTO users (name, email, password, role) VALUES (?, ?, ?, ?)`, 
        ['Administrador', 'admin@ong.com', adminPassword, 'admin']);
});

// Rotas da API
// (Implementar todas as rotas conforme especificado no código completo)

app.listen(PORT, () => {
    console.log(`Servidor rodando em http://localhost:${PORT}`);
});

10. Guia de Uso
10.1 Gerenciar Eventos
Criar Evento
1.	Acesse a aba "Eventos"
2.	Preencha os campos:
o	Título (obrigatório)
o	Descrição
o	Data e Hora (obrigatório)
o	Local
o	Imagens (múltiplas, a primeira será capa)
o	Status
3.	Clique em "Criar Evento"
Editar Evento
1.	Localize o evento na lista
2.	Clique em "Editar"
3.	Altere as informações desejadas
4.	Para remover imagens, clique no X sobre a imagem
5.	Adicione novas imagens se necessário
6.	Clique em "Salvar Alterações"
Excluir Evento
1.	Localize o evento na lista
2.	Clique em "Excluir"
3.	Confirme a exclusão
10.2 Gerenciar Finanças
Registrar Movimentação
1.	Acesse a aba "Finanças"
2.	Selecione o tipo (Receita ou Despesa)
3.	Escolha a categoria
4.	Preencha descrição (opcional)
5.	Informe o valor (use ponto para decimais, ex: 99.90)
6.	Selecione a data
7.	Clique em "Registrar"
Visualizar Relatórios
•	Os totais são atualizados automaticamente
•	O gráfico de despesas por categoria é gerado dinamicamente
•	A lista de lançamentos mostra todos os registros
Excluir Lançamento
1.	Localize o registro na lista de lançamentos
2.	Clique em "Excluir"
3.	Confirme a exclusão
10.3 Gerenciar Usuários
Criar Usuário
1.	Acesse a aba "Usuários"
2.	Preencha:
o	Nome completo
o	Email
o	Senha
o	Nível (Admin ou Editor)
3.	Clique em "Criar Usuário"
Editar Nível de Acesso
1.	Localize o usuário na lista
2.	Clique em "Editar"
3.	Confirme a alteração no popup
Excluir Usuário
1.	Localize o usuário na lista
2.	Clique em "Excluir"
3.	Confirme a exclusão
Nota: O usuário administrador principal (ID 1) não pode ser excluído.

11. Solução de Problemas
11.1 Erro: "totalReceita.toFixed is not a function"
Causa: Valores financeiros retornando como string ou null.
Solução: O código corrigido já trata este problema. Certifique-se de usar a versão mais recente do frontend.
11.2 Erro de Conexão com API
Verificar:
bash
# Testar se o servidor está rodando
curl http://localhost:3000/api/events

# Verificar porta ocupada
netstat -ano | findstr :3000
Soluções:
•	Confirme que o backend está rodando
•	Verifique se a porta 3000 está livre
•	No frontend, confirme a URL do API_URL
11.3 Erro de Autenticação
Verificar:
•	Credenciais corretas
•	Token JWT não expirado
•	Limpar localStorage e tentar novamente
javascript
// Limpar token manualmente no console
localStorage.clear();
location.reload();
11.4 Upload de Imagens Não Funciona
Verificar:
•	Pasta uploads/ existe e tem permissão de escrita
•	Tamanho da imagem não excede 5MB
•	Formato da imagem é jpg, png ou gif
11.5 Banco de Dados Corrompido
Recuperar:
bash
# Backup do banco atual
cp database.sqlite database.sqlite.backup

# Recriar banco (remover e iniciar servidor)
rm database.sqlite
npm start  # Recriará as tabelas

12. Manutenção e Backup
12.1 Backup do Banco de Dados
bash
# Backup manual
cp backend/database.sqlite backups/database_$(date +%Y%m%d).sqlite

# Script de backup automático (Linux/Mac)
#!/bin/bash
mkdir -p backups
cp backend/database.sqlite backups/database_$(date +%Y%m%d_%H%M%S).sqlite
find backups -name "database_*.sqlite" -mtime +30 -delete
12.2 Backup das Imagens
bash
# Backup da pasta uploads
tar -czf uploads_backup_$(date +%Y%m%d).tar.gz backend/uploads/
12.3 Logs do Sistema
Adicione logging no server.js para monitoramento:
javascript
// Middleware de logging
app.use((req, res, next) => {
    console.log(`${new Date().toISOString()} - ${req.method} ${req.url}`);
    next();
});
